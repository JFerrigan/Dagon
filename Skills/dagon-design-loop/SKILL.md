# Dagon Design Loop

## Overview

Use this skill for back-and-forth design work on Dagon. It keeps design discussion anchored to the existing planning docs and pushes toward concrete decisions instead of broad brainstorming drift.

## Workflow

1. Read [Docs/GamePlan.md](/Users/jakeferrigan/Echo Rift/Docs/GamePlan.md).
2. Identify whether the request is about:
   - fiction and tone
   - combat and progression
   - biome and enemy design
   - scope and MVP
3. Respond with specific recommendations and tradeoffs.
4. When decisions are made, write them back into the docs.
5. Keep new ideas consistent with the current anchors:
   - sailor protagonist
   - black mire opening biome
   - Dagon / maritime cosmic horror
   - corruption as a defining mechanic

## Preferred Output Style

- concrete suggestions over open-ended lists
- gameplay implications called out explicitly
- MVP-safe recommendations by default

## When To Read References

- Read `references/design-anchors.md` when a request risks drifting away from the established direction.

## Boundaries

- Do not broaden the scope casually.
- Do not introduce mechanics that undermine the bullet-heaven core unless the user asks.
- Treat corruption as a central differentiator, not a cosmetic theme.
